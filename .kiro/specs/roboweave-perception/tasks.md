# Implementation Plan: roboweave_perception

## Overview

Build the `roboweave_perception` ROS2 ament_python package following the same conventions as `roboweave_control`. Implementation proceeds bottom-up: abstract interfaces → backend registry → mock/simple backends → converters → pose tracker → perception node → config & launch. Property-based tests (Hypothesis) and unit tests are interleaved with implementation tasks to catch errors early.

## Notes

- Tasks marked with `*` are optional and can be skipped for faster MVP

- Checkpoints ensure incremental validation
- Property tests validate the 10 universal correctness properties from the design document
- Unit tests validate specific examples and edge cases
- All backends and converters are pure Python (no ROS2 required for testing)
- The `HAS_ROS2` conditional pattern enables testing without a ROS2 workspace
